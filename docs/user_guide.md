# 📖 Guia do Comandante: Sovereign Pair (v1.3.x)

Bem-vindo ao centro de comando da sua soberania digital. Este guia detalha como operar as abas e motores do Sovereign Pair para extrair o máximo de inteligência cíbrida.

---

## 1. A Filosofia do Controle

Diferente de assistentes comerciais, aqui você não é o "produto". Você é o **Comandante**. Cada clique na interface invoca processos nativos, scripts sandbox e túneis SSH sob sua supervisão direta.

---

## 2. Interface e Navegação

### 2.1 Chat & RAG Router

O portal de interação principal.

- **Modo Cibrid**: Alterna dinamicamente entre inferência local (privacidade total) e offload em nuvem (performance extrema).
- **Injeção de Contexto (Vault)**: Se você estiver lendo um documento no Vault e abrir o chat, o sistema injeta automaticamente o contexto do arquivo para respostas ultra-precisas.

### 2.2 Deep Research & Tool Calling

Ative o **Deep Research** para investigações complexas.

- **Scribe & Auditor**: O motor gera um relatório técnico (Scribe) que é imediatamente auditado por um modelo superior (Auditor) para garantir que nenhum dado foi inventado.
- **Fact Verification**: Cada número citado deve ter uma fonte rastreável no Vault ou na Web.

---

## 3. Vault & Gestão de Conhecimento

O Vault é seu repositório físico de arquivos Markdown, PDFs processados e Segredos Criptográficos.

- **Unified SecOps Vault**: A aba de segurança consolida todas as suas credenciais. Se você deseja rotear IA para a nuvem, simplesmente insira uma chave do tipo `API_KEY` com o nome do provedor (ex: `QWEN_KEY`, `NVIDIA_KEY`, `OPENROUTER_KEY`) e o Sovereign interceptará a configuração silenciosamente sob o capô. Você também pode inserir chaves SSH para conectar a instâncias Oracle Cloud.
- **Sensus Sync**: Toda nota criada na UI é salva instantaneamente como um arquivo `.md` no seu HD. Você é dono dos seus dados, sem formatos proprietários.
- **Grounding Engine**: Ao ativar o "Strict Grounding", a IA ignora o conhecimento de treinamento e responde **apenas** com base nos seus documentos.

---

## 4. Telemetria e Hardware

Acompanhe em tempo real o que acontece "sob o capô":

- **Tokens per Second (T/s)**: Velocidade de leitura da rede neural.
- **VRAM Monitor**: Vital para evitar estouros de memória. Se a VRAM estiver cheia, o **OOM Guard** reduzirá automaticamente a janela de contexto.
- **Resilience Shield**: Monitora a saúde das suas APIs externas (NVIDIA, Qwen, etc) e alerta sobre quedas de serviço.

---

## 5. Roadmap de Funcionalidades (Pavimentação v1.4.0)

Para manter a transparência técnica, identificamos as áreas que estão em fase de "Cenografia Estrutural" (Mocks) preparando o terreno para as próximas implementações:

1. **Fine-Tuning Engine (Simulado)**: A interface de ajuste de pesos (LoRA/Alpha) está pronta, mas o motor de retreino via Python VRAM está sendo integrado para a v1.4.0. Atualmente, o sistema simula a telemetria para validação de UX.
2. **Reflection Lab (Simulado)**: O laboratório de metacognição gera auditorias randômicas para testar o fluxo de raciocínio latente que será nativo em breve.

---

## 6. Dicas de Performance

- **Use o Reranker**: Para bases de conhecimento grandes, ative o _Cross-Encoder_. Ele limpa o "ruído" e entrega apenas os melhores fatos para o modelo.
- **Nuvem Híbrida**: Use o Nó Oracle para tarefas pesadas e o Nó Local (Ollama) para conversas rápidas e sensíveis.
